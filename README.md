# Blind Runner

A 3D stealth-horror game built in Unity. The player is trapped in a fully dark, procedurally generated maze and can only see the world through echolocation — a sound ping that briefly lights up walls and enemies with an expanding ring of light. Every ping is audible, and enemies move toward the sound, creating a constant tension between "look around" and "give yourself away."

## Concept

The world is completely black, with no conventional lighting at all. Pressing space launches a ring of light that expands outward from the player, illuminating any geometry it crosses and leaving a fading afterglow behind. Enemies stay invisible until the ping reveals them, and they listen for noise generated by the player — quieter while sneaking, louder while running, loudest of all from the ping itself.

## Technology

- Unity 6 (URP)
- Custom HLSL shader for the echolocation effect
- NavMesh + NavMeshSurface (rebaked dynamically for procedural levels)
- ProBuilder (geometry prototyping)
- Post Processing (Bloom, Vignette)

## Implemented Features

**Player**
- Movement with three speeds (sneak / walk / run), direction relative to the camera
- Third-person camera with auto-realignment behind the character and a manual reset key
- Noise system — every action generates its own hearing radius

**Echolocation**
- Custom HLSL shader: only lights up pixels at the current ring radius, everything else stays black
- Fading afterglow trail left behind after the wave passes
- Separate color coding for walls (blue) and enemies (red)
- Cooldown with visual readiness indicator on the HUD

**Enemies**
- Finite state machine: patrol → investigate sound → chase
- NavMeshAgent for navigation, hear the player based on noise radius
- Animations and audio (footsteps, detection sound on entering chase)

**Levels**
- Procedural maze generation (Recursive Backtracking)
- Guaranteed path from start to exit
- Multiple levels with increasing enemy count
- Full game loop: main menu → levels → victory/death → pause

## Project Structure

```
Assets/
├── Scripts/      — all gameplay logic
├── Shaders/       — echolocation shader (separate variants for walls and enemies)
├── Prefabs/       — player, enemy, maze elements
├── Materials/      — materials using the echolocation shader
├── Animations/      — Animator Controllers for player and enemy
├── Audio/         — footstep, ping, and detection sounds
├── UI/           — main menu, HUD, pause/victory/death screens
└── Scenes/        — MainMenu + levels
```

## Status

A student project developed as part of a computer graphics course. The prototype is fully playable from start to victory/death.
