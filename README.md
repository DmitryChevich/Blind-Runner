# Blind Runner

3D стелс-хоррор на Unity. Игрок заперт в полностью тёмном процедурном лабиринте и видит мир только через эхолокацию — звуковой пинг, который на мгновение подсвечивает стены и врагов кольцом света. Каждый пинг слышен, и враги идут на звук — баланс между "посмотреть" и "выдать себя" это и есть игра.

## Концепция

Мир абсолютно чёрный, без какого-либо обычного освещения. Нажатие пробела запускает кольцо света, расходящееся от игрока — оно подсвечивает геометрию которую пересекает и оставляет угасающий след. Враги невидимы до тех пор, пока их не задел пинг, и слышат шум который производит игрок: тише при крадении, громче при беге, громче всего при самом пинге.

## Технологии

- Unity 6 (URP)
- Shader Graph / собственный HLSL шейдер для эффекта эхолокации
- NavMesh + NavMeshSurface (динамическая перепечка для процедурных уровней)
- ProBuilder (заготовки геометрии)
- Post Processing (Bloom, Vignette)

## Реализованные механики

**Игрок**
- Передвижение с тремя скоростями (крадётся / идёт / бежит), направление относительно камеры
- Камера от третьего лица с авто-разворотом за спину персонажа и сбросом по клавише
- Система шума — каждое действие генерирует свой радиус слышимости

**Эхолокация**
- Кастомный HLSL шейдер: подсвечивает только пиксели на текущем радиусе кольца, остальное остаётся чёрным
- Угасающий след после прохождения волны
- Отдельная цветовая кодировка для стен (синий) и врагов (красный)
- Кулдаун и визуальная индикация готовности на HUD

**Враги**
- Конечный автомат: патруль → расследование звука → погоня
- NavMeshAgent для навигации, слышат игрока по радиусу шума
- Анимации и звуки (шаги, звук обнаружения при переходе в погоню)

**Уровни**
- Процедурная генерация лабиринта (Recursive Backtracking)
- Гарантированная проходимость старт → выход
- Несколько уровней с нарастающим числом врагов
- Полный игровой цикл: главное меню → уровни → победа/смерть → пауза

## Структура проекта

```
Assets/
├── Scripts/      — вся игровая логика
├── Shaders/       — шейдер эхолокации (свой для стен, свой для врагов)
├── Prefabs/       — игрок, враг, элементы лабиринта
├── Materials/      — материалы с шейдером эхолокации
├── Animations/      — Animator Controller для игрока и врага
├── Audio/         — звуки шагов, пинга, обнаружения
├── UI/           — главное меню, HUD, экраны паузы/победы/смерти
└── Scenes/        — MainMenu + уровни
```

## Статус

Учебный проект, разрабатывается в рамках курса по компьютерной графике. Прототип полностью играбелен от старта до победы/смерти.
